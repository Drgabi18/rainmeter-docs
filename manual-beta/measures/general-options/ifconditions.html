--- layout: manual-beta title: 'IfConditions' ---

IfCondition options are used in any measure, of any type, to evaluate a mathematical formula you define. The formula will be evaluated as "true" or "false", and will execute IfTrueAction or IfFalseAction action options, which will contain one or more Bangs or commands. The IfTrueAction or IfFalseAction options will be executed on each update of the measure, based on the formula evaluating, or continuing to evaluate, as "true" or "false".

There can be as many sets of IfCondition and IfTrueAction / IfFalseAction options as desired, by using IfCondition2, IfCondition3 and so on. For each IfConditionN option in the measure, there can be one instance of either IfTrueActionN, IfFalseActionN or both.

The formula that is used in the IfCondition option can be any mathematical formula, using the same syntax and operators described for the Calc measure. The only Calc measure specific functions that can't be used are Counter and Random.

Generally speaking, IfCondition will be used to create a test using the current value of one or more measures, and take action when the formula either evaluates as "true" or "false". This is similar in concept to the IfActions functionality, with a few important and powerful differences.

  • The formula is not based on the value of the measure the IfCondition is in, but can reference any measures, variables or section variables.
  • Since there can be an IfTrueActionN, IfFalseActionN or both for each IfConditionN, you can take appropriate actions not only when the result is "true", but also when it is "false".
  • The IfTrueAction or IfFalseAction is executed on each update of the measure. Unlike IfActions, it is not dependent on the IfCondition being "true" then "false" then "true" again.
  • The syntax allows for a single IfCondition to embed logical AND && and OR || comparisons in the formula. Unlike IfEqualValue for instance, which allows for a single comparison, you could use IfCondition=(MeasureName = 5) || (MeasureName = 10), to check if the value of MeasureName is "5" OR "10". You can also use the full range of logical operators, so rather than being limited to "equal", "above" or "below", you can use any of <> = > < <= >= && ||.

    • Options

    IfCondition, IfCondition2, IfCondition3 ...

    A mathematical formula, which when evaluated will result in "true" or "false".

    Examples:

    IfCondition=MeasureName >= 10

    IfCondition=MeasureOne = ((MeasureTwo / 5) + 1)

    IfCondition=(MeasureName > 5) && (MeasureName < 10)

    IfCondition=(MeasureName = 25) || (MeasureName = 50) || (MeasureName = 75)

    Note: The comparison on either side of the logical AND && and OR || operators must be enclosed in (parentheses).

    IfTrueAction, IfTrueAction2, IfTrueAction3 ...

    Optional Action to be executed when the formula expressed in IfCondition results in "true".

    IfFalseAction, IfFalseAction2, IfFalseAction3 ...

    Optional Action to be executed when the formula expressed in IfCondition results in "false".

    Note: Since the IfCondition is evaluated on every update of the measure, care should be taken not to create an infinite loop with the actions taken in IfTrueAction or IfFalseAction. In particular, !Refresh, !Update and !UpdateMeasure #CURRENTSECTION# should be avoided.

    Example

    {% highlight ini %} [Rainmeter] Update=1000 DynamicWindowSize=1 AccurateText=1 [MeasureCPU] Measure=CPU IfCondition=(MeasureCPU < 10) IfTrueAction=[!SetOption MeterCPU Text "CPU usage is less than 10 percent"] IfCondition2=(MeasureCPU >= 10) && (MeasureCPU <= 90) IfTrueAction2=[!SetOption MeterCPU Text "CPU usage is between 10 and 90 percent"] IfCondition3=(MeasureCPU > 90) IfTrueAction3=[!SetOption MeterCPU Text "CPU usage is more than 90 percent!"] OnUpdateAction=[!UpdateMeter MeterCPU][!Redraw] [MeterCPU] Meter=String FontSize=12 FontColor=255,255,255,255 SolidColor=47,47,47,255 Padding=5,5,5,5 AntiAlias=1 {% endhighlight %}
