--- layout: manual-beta title: 'FileView plugin' ---

Plugin=FileView retrieves information about folders and files.

The plugin gathers all the folder and file names, sizes, dates and icons in the selected folder. It counts the number of files and folders and obtains the combined size. It can also search recursively through all the subfolders of the selected folder to obtain the totals for count and size.

FileView operates with a "parent / child" approach. A main "parent" FileView measure is used to obtain all the information for a selected folder, and then "child" measures are used to read individual entries from the parent using the Path= option.

Usage

FileView measures take the form:

{% highlight ini %} [Rainmeter] Update=1000 DynamicWindowSize=1 [MeasureFolder] Measure=Plugin Plugin=FileView Path="C:\Program Files\Rainmeter" ShowDotDot=0 ShowFolder=0 Count=3 {% endhighlight %}

In this example, this "parent" measure will obtain name, size, date and icon information about all files in the selected folder, then create three Index values for the first three files. The information is used in subsequent "child" FileView measures:

{% highlight ini %} [MeasureChild1] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileName Index=1 [MeasureChild2] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileName Index=2 [MeasureChild3] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileName Index=3 {% endhighlight %}

The values of the three child measures are now the FileName information parsed into Indexes 1 through 3 by the parent measure. These can then be used with MeasureName and other options in meters.

Another way to use the information in child measures is:

{% highlight ini %} [MeasureChild1] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileName Index=1 [MeasureChild2] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileSize Index=1 [MeasureChild3] Measure=Plugin Plugin=FileView Path=[MeasureFolder] Type=FileDate Index=1 {% endhighlight %}

The values of the three child measures are now the FileName, FileSize and FileDate information from the first file (Index 1) of the parent measure.

In these examples, three indexes are created due to the Count=3 option on the parent measure. However, the information for all files and / or folders are obtained by the plugin, and subsequent or previous entries can be assigned to the three indexes by using !CommandMeasure statements described below. This will allow a skin to "scroll" through the files and / or folders in a dynamic way.

Options

General measure options

All general measure options are valid.

Path

In a parent measure, the path of the folder to read. If Path="", then the plugin defaults to My Computer.
In a child measure, this is used to identfy the [MeasureName] of the parent measure to read values from.

Parent measure only options

Recursive Default: 0

If 1, the plugin searches all sub-folders updating only the file count, folder count and overall folder size. This option does not index specific files in sub-folders.

Count Default: 1

The number of items to be indexed at one time.

ShowDotDot Default: 1

If 1, the .. folder (representing the previous folder) will be included. ShowDotDot=0 means that the .. folder is ignored.

ShowFolder Default: 1

If 1, folders will be included, along with files. If ShowFolder=0, folders are ignored.

Note: This does not control the .. folder, use ShowDotDot above.

ShowFile Default: 1

If 1, files will be included, along with folders. If ShowFile=0, files are ignored.

ShowHidden Default: 1

If 1, hidden files and folders will be included. If ShowHidden=0, hidden files and folders are ignored.

ShowSystem Default: 0

If 0, protected operating system files are ignored. If ShowSystem=1, protected operating system files are included.

HideExtensions Default: 0

If 1, file extensions are removed when used with Type=FileName.

Extensions

Semi-colon separated list of file extensions that limits the type of files to be included. (eg. If Extensions="jpg;png", only .jpg and .png files will be included.

SortType Default: Name

Type of information to sort the entries by. Options include: Name, Size, Type, Date.

SortDateType Default: Modified

File and folder date entry to use for sorting when SortType=Date. Options include: Modified, Created, Accessed.

SortAscending Default: 1

If 1, sorts the entries in ascending order. SortAscending=0 will sort in descending order.

WildcardSearch Default: "*.*"

Filter the files and folders obtained using wildcards. Standard * and ? characters can be used.

Child measure only options

Index Default: 1

Index of the file or folder from the parent measure. This should not exceed the Count number in the parent measure. If it does, the items will wrap around. So if Count=8, then Index=9 will return the value from Index=1.

IgnoreCount Default: 0

If 1, the Index will represent the actual index of the file or folder in the list. This is useful to display a particular file or folder at all times (like the .. folder).

Type Default: FolderPath

Type of information to obtain from the parent measure. Options include: FolderPath, FolderSize, FileCount, FolderCount, FileName, FileType, FileSize, FileDate, FilePath, and Icon.

Note: If Type=Icon, the the full path and name of the icon file will be the returned value See IconPath below. This can be used with MeasureName or ImageName in an Image meter to display the icon.

DateType Default: Modified

The date entry to retrieve from the parent measure when Type=Date. Options include: Modified, Created, Accessed.

IconPath

Path and name of file location where the icon is to be saved when Type=Icon. If no path is given, the icon will be saved in the current skin folder in the style of "iconX.ico", where "X" is the index number.

IconSize Default: Medium

Size of the icon to save. Options include: Small for a 16x16 icon, Medium for a 32x32 icon, Large for a 48x48 icon, ExtraLarge for a 256x256 icon.

Plugin Commands

Parent measure only commands

PageUp

Decrease the current page count. For instance, If Count=8, and there are currently 25 files in the list, then there are a total of 4 pages. So if the plugin is currently displaying items 8-15 (page 2), and the PageUp command is sent to the parent measure, it will decrease the page count by one, and the items that will be displayed will be items 0-7 (page 1).

Usage example: LeftMouseUpAction=!CommandMeasure "ParentMeasureName" "PageUp".

PageDown

PageDown - Increases the page count.

Usage example: LeftMouseUpAction=!CommandMeasure "ParentMeasureName" "PageDown".

IndexUp

Decreases the index by 1. This is useful for mouse scroll actions.

Usage example: MouseScrollUpAction=!CommandMeasure "ParentMeasureName" "IndexUp".

IndexDown

Increases the index by 1. This is useful for mouse scroll actions.

Usage example: MouseScrollDownAction=!CommandMeasure "ParentMeasureName" "IndexDown".

Update

Updates the measure, reading the disk and recreating all values in the parent measure.

Usage example: LeftMouseUpAction=!CommandMeasure "ParentMeasureName" "Update".

PreviousFolder

This will change the path to the folder one higher in the folder structure. (behaves like clicking on the .. folder.)

Usage example: LeftMouseUpAction=!CommandMeasure "ParentMeasureName" "PreviousFolder".

Child measure only commands

FollowPath

If the index referenced in the child measure is currently a folder, then the parent measure's Path is updated to the new path. If it is a file, then it is opened with the default Windows associated application. In order to better simulate Windows behavior, it is recommended that this command be used with a "DoubleClick" action.

Usage example: LeftMouseDoubleClickAction=!CommandMeasure "ParentMeasureName" "FollowPath".

Open

This will open the file or folder that the index represents. If it is a folder, the folder is opened in Windows Explorer. If it is a file, then it is opened with the default Windows associated application. In order to better simulate Windows behavior, it is recommended that this command be used with a "DoubleClick" action.

Usage example: LeftMouseDoubleClickAction=!CommandMeasure "ParentMeasureName" "Open".
