This tutorial is part of a series. If you aren't sure how you got here, you may want to start from the beginning.

Now that you have a solid grasp on meters, it's time to introduce you to the "other half" of skins: measures. Unlike meters, measures aren't things that can be seen or clicked. They exist only in the background. The purpose of measures is to gather information from some source outside of Rainmeter: from Windows, another application, a file on your computer, or even a webpage on the Internet.

How you use this information is up to you. But usually, the reason for having a measure is to display the value using a meter.

Before we go further, let's clean up your skin a little bit. Remove [MyMeter2] and [MyMeter3], and the Y=38r option from your string style.

{% highlight ini %} [Rainmeter] Update=1000 [MyStringStyle] AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 SolidColor=0,0,0,1 W=320 H=38 [MyMeter] Meter=String MeterStyle=MyStringStyle Text=Hello, world! {% endhighlight %}

Time Measure

Let's look at one of the simplest measures: the time measure. Yes, you're about to create another one of Rainmeter's infamous clock skins.

{% highlight ini %} [MyMeasure] Measure=Time {% endhighlight %}

Just like the Meter option for meters, the Measure option tells Rainmeter that this section is a measure, and the value sets the specific type of measure. Unlike some other measures, the time measure does not require any additional options—it will start giving us useful information right away.

To see that the measure is now working, refresh the skin, then open the About window. You can do this by clicking Open log in the Manage window, or About in the tray icon's context menu. Then, click the Skins tab.
