--- layout: manual title: 'Meters Tutorial' ---

This tutorial is part of a series. If you aren't sure how you got here, you may want to start from the beginning.

Let's get started! This tutorial will focus on creating your first meters. Meters are a skin's display elements: the parts of the skin that you can see and click on your desktop. Every skin must have at least one meter, so that's where we'll begin.

Hopefully, you already have the MyFirstSkin.ini file that you created earlier. Open the skin in your text editor.

[Rainmeter]

The very first thing you're going to add to your skin is the [Rainmeter] section. This is a skin's "header" property, like the <head> tag in an HTML webpage. For now, your [Rainmeter] section will be mostly empty, except for one option:

{% highlight ini %} [Rainmeter] Update=1000 {% endhighlight %}

The Update option is what sets the length of the skin's update cycle. The length is given in milliseconds, or 1/1000ths of a second, so Update=1000 means that the skin will update once per second. Updating is how the skin will react to changes in information. You'll see how this works in more detail later on.

String Meter

Now that you've given your skin a "head," it's time to give it a "body." You're going to create a string meter. This is one of the most common types of meters, and it is used to create text.

{% highlight ini %} [Rainmeter] Update=1000 [MyMeter] Meter=String Text=Hello, world! {% endhighlight %}

The Meter option is required to tell Rainmeter that this section is, in fact, a meter. All meters have this option. The value of the option determines what type of meter it is.

The Text option, on the other hand, is unique to the string meter. As you might have guessed, this is where you provide a string of text for Rainmeter to display.

Believe it or not, what you have now is a complete, valid, working Rainmeter skin! Let's load it to see what it looks like. Load the skin using one of the methods that you learned before. You can either:

  • Open the Manage window by left-clicking the Rainmeter tray icon, find MyFirstSkin in the skins list, then click the Load button in the upper-right.

  • Open the context menu by right-clicking the tray icon, then select Skins → MyFirstSkin → MyFirstSkin.ini.

(As you get comfortable with Rainmeter's user interface, you'll decide whether you prefer working with the context menu or the Manage window.)

Now, look up in the top-left corner of your desktop. There's your skin!

Can you see me?

It's... not very big. Or pretty. Meters without any options tend to be very simple and unimpressive. So let's add some formatting.

{% highlight ini %} [MyMeter] Meter=String Text=Hello, world! AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 {% endhighlight %}

Here's what we've added:

  • AntiAlias
    A general meter option that smooths out the edges of a meter. This almost always improves the appearance of a string meter.

  • FontColor
    A color option that changes the color of the text in this meter.

  • FontFace
    An option that changes the font used for this meter. Rainmeter can use any font that you have installed in Windows, or another font in a skin's @Resources folder—but we'll get to that.

  • FontSize
    The size of the font.

Now, let's apply these changes by refreshing the skin. Once again, you can either press Refresh in the Manage window, or MyFirstSkin → Refresh skin in the context menu.

Much nicer.

Transparency

Now that your skin is looking a little nicer, why don't you try clicking and dragging it somewhere else on your desktop?

You may have found it hard to click. That's because Rainmeter skins are only clickable in a spot where there are some visible pixels. Right now, the only visible pixels are the letters themselves. The space in between the letters is completely transparent: there are no visible elements.

To rectify this, we're going to add a SolidColor option. This is another color option that you can use to add a colored background to the entire area of any meter.

{% highlight ini %} [MyMeter] Meter=String Text=Hello, world! AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 SolidColor=0,0,0 {% endhighlight %}

Contrast FTW.

With the SolidColor option, it's also easier to see that the string meter, like all meters, has a perfectly rectangular area, with a defined width and height. This will be important later on, when you start working with your meters' positions and dimensions.

For now, though, the skin looks kind of ugly with all that black, doesn't it? Let's try taking it down a little by adding some partial transparency to the SolidColor option. Adding a fourth term to any color option lets you change the opacity of the element, on a scale of 0–255:

SolidColor=0,0,0,128

I'm melting...

Hm. Better. But all we wanted to do was make the skin easier to click and drag, right? So let's take it down as far as it can go:

SolidColor=0,0,0,1

We hardly knew you, SolidColor.

Now, the background is almost perfectly invisible. But since it's still technically non-transparent—with an opacity of exactly 1/255—we can now click any part of the skin. Success!

Position

Let's try adding another meter.

{% highlight ini %} [MyMeter] Meter=String Text=Hello, world! AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 SolidColor=0,0,0,1 [MyMeter2] Meter=String Text=I'm another meter. AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 SolidColor=0,0,0,1 {% endhighlight %}

What does it say?

Well, that's an issue. The reason that these meters are overlapping is because neither of them has a position. As a result, they both start in the same part of the skin window: specifically, the top-left corner.

How do we move the second meter? First, we need to understand the X and Y coordinate system.

Absolute Positions

Since Rainmeter skins are two-dimensional, we use two numbers to place meters. X is the number of pixels from the leftmost edge of the skin to the meter; Y is the number of pixels from the topmost edge. All meters start at X=0 and Y=0 by default; this is called the origin of the skin. So in order to move this meter down, we need to add a Y option:

Y=38

This pushes the second meter down 38 pixels. Now, both meters can be read clearly:

Relative Positions

What if we add a third meter?

{% highlight ini %} [MyMeter3] Meter=String Text=Yet another string meter. AntiAlias=1 FontColor=255,255,255 FontFace=Segoe UI FontSize=20 SolidColor=0,0,0,1 Y=76 {% endhighlight %}

Now we have:

  • [MyMeter] with no Y=0 (the default).
  • [MyMeter2] with Y=38.
  • [MyMeter3] with Y=76.

This works great. But if we start adding even more meters, it's going to get awfully tedious to calculate the new Y values by hand - especially if we add meters with different sizes. To solve this problem, Rainmeter allows you to place meters relative to the position of the previous meter, no matter where it is—you don't need to know the exact number. Putting a lowercase r on the end of the Y option changes the starting point from the origin of the skin to the origin of the previous meter. Using this feature, placing our meters becomes much more straightforward:

  • [MyMeter2] with Y=38r.
  • [MyMeter3] with Y=38r.

The skin looks the same as it did before. But this gives an advantage to you as the skin author, making your code much easier to manage, especially if you want to change something later.

Dimensions

As we discovered earlier, meters have not only positions, but dimensions. Let's change the opacity on our SolidColor option again, so that we can see the dimensions of our meters more easily:

SolidColor=0,0,0,128

Because each string is a different length, each meter has a different width. This makes the skin look uneven. However, by adding width and height options, we can force these meters into a certain shape. Just as X sets a number of pixels from the left edge of the skin, W (width) makes the meter extend a number of pixels from the left.1 Likewise, H (height) makes the meter extend a number of pixels from the top. So if we add both2 width and height options to all three meters, their areas will line up exactly.

Add these options to [MyMeter], [MyMeter2] and [MyMeter3]:

W=320
H=38

  1. There is an exception to this rule. String meters with a StringAlign option may change their origins from the top-left to the right, center, and/or bottom of the meter.
  2. Most meter types require both a W and H option in order to set the dimensions. If one is missing, the other is ignored.
