Troubleshoot

Oftentimes, you may run into problems while using Rainmeter. This page tries to detail the most common problems and solutions for how to fix them.

Since Rainmeter is ever-changing, problems and solutions on this page may change or be removed, make sure you are using the latest version, as that is what the solutions on this page assume.

Rainmeter will run on Windows 7 (Service Pack 1 and Platform Update required) and above. .NET Framework 4.7.1 and above needs to be installed to use Rainmeter, the last update for Windows 7, as well as Windows 8 and above, have the .NET Framework pre-installed.

Occasionally when Rainmeter is starting up, it will crash immediately (or after a period of freezing).

This can happen due to various reasons, including:

  1. Attempting to load a "bad" skin - which may contain things such as infinite update loops, actual Rainmeter bugs, broken plugins etc.
  2. Windows problems - this includes faulty Windows Installation, faulty .NET installations, faulty drivers, corrupted Performance Monitor counters database, missing winlanapi.dll dependency (only on Windows Server) etc.
  3. The setting config file gets corrupted
  4. Rainmeter problems - faults in the code that lead to it crashing

If this is happening, a solution is opening Rainmeter using Safe Start. You can hold down the CTRL key while running the shortcut or .exe for Rainmeter, and it will offer to Safe Start Rainmeter.

What this will do is unload all active skins, and load the default Illustro Layout that comes with Rainmeter.

This does not remove or in any way harm the skins you have, but simply loads the default Illustro Layout. Your current setup will be saved as a Layout in @Backup.

You may spot a skin looking blurry, or a skin set to be sized exactly your display resolution but ended being larger than you thought.

This is a Windows issue caused by the default DPI setting.

By default, Windows set your DPI to 125%, which makes all Windows Forms bigger, unless specified not to on an individual app basis.

Regarding whether you want to change your system-wide DPI or Rainmeter-specific DPI, you can either:

  • Change the Rainmeter Specific DPI

    • To change Rainmeter to ignore this and use 100% DPI, find the "shortcut" for Rainmeter, which you will find in your Startup folder in Windows:

    Right-click that shortcut, and select "Properties". Under Properties, select the "Compatibility" tab, and do this:

    Restart Rainmeter using the shortcut.

    Note that there are a couple of places in Rainmeter that won't be effected by this, since they are things that are not actually created and rendered by Rainmeter, but by Windows itself. These are the FontSize in InputText fields, and the size of context menus in skins. These will always use the DPI setting in Windows.

  • Change system-wide DPI

    • The setting for DPI is done in the Setting panels in Windows.

Windows updates and decisions Microsoft takes are causing more problems with installing Rainmeter skins. Here are some of the main culprits:

  • Controlled Folder Access

    • In order to react to the recent bout of "ransomware" attacks, where some malware encrypts some of your personal files, and offers to unencrypt them for a payment, Windows has added a new optional feature, Controlled Folder Access.

    What this does is "protect" certain personal folders that have been the target of these "ransomware" attacks like: Documents, Music, Videos, Pictures, Favorites, Desktop.

    If this feature is turned on, no applications will be allowed to write to these folders, unless they are "whitelisted" in the security settings for the feature. There are some well trusted applications (almost all of them those owned by Microsoft itself) that are automatically whitelisted, but all others will need to be added if you want them to be able to write to these folders.

    In order to use Rainmeter with this feature turned on, the following program files will need to be whitelisted:

    • C:\Program Files\Rainmeter\Rainmeter.exe

    • C:\Program Files\Rainmeter\SkinInstaller.exe

    Otherwise, Rainmeter.exe will not be able to write to the ..Documents\Rainmeter\Skins folder, and will be unusable. In addition, you must whitelist SkinInstaller.exe or it will not be able to install .rmskins to your skins folder, and you will not be able to save .rmskin files you create to the default Desktop location.

  • OneDrive

    • Windows 10 defaulted the User Profile folders to the OneDrive storage folder, while this may be great for personal use, the OneDrive folder revokes write/read permission during syncing or "when phoning home" and stops Rainmeter from creating / modifying skin files and folders.

    To rectify this, you need to add/change the SkinPath option, inside the settings file.

    To get to the settings file:

    • From inside Rainmeter: Open the Manage Window or right click the Tray Icon, click Edit Settings.
    • From outside Rainmeter: Press WIN+R and type %AppData%\Rainmeter\rainmeter.ini to open the Settings file.

    Now, either modify the existing value for that option or add it under [Rainmeter]. The path should not be another folder on OneDrive or any Program Files folder on any of the volumes.

When you click either of those buttons, Rainmeter searches for the executable associated with the .ini file format and attempts to launch it. If the executable exist inside a protected folder (which it dose in Windows 11), Rainmeter's attempt to launch the executable errors out due to a lack of permissions.

You need to change the ConfigEditor option.

To do so, right click the Rainmeter icon in your system tray, then select Manage. Navigate to the Settings tab, and next to Editor, selet "..." and choose a different editor, either choose C:\WINDOWS\system32\notepad.exe or an editor of your liking.

While rare, it is possible for the database of counters that is maintained by Performance Monitor in Windows to become damaged, which will cause the UsageMonitor plugin to stop functioning correctly.

In many cases, this can be corrected by "rebuilding" the database from the Windows command line.

Run cmd.exe, being sure to run "As administrator". At the command line, enter the following commands in order:

``` dos cd c:\windows\system32 lodctr /R cd c:\windows\sysWOW64 lodctr /R ```

You should see the following results:

Info: Successfully rebuilt performance counter setting from system backup store

There are multiple reasons why this may appear:

  1. Your Rainmeter installation has the wrong bitness (running 64bit install on a 32bit System)
  2. Your C++ libraries are missing/corrupted - to fix this, make sure you have installed the correct Microsoft Visual C++ Redistributable architecture for your system

  3. You have 2 Rainmeter instances in the list of apps when Windows Starts Up
  • If Rainmeter crashes without warning

    • Use the Event Viewer, which usually logs events that have taken place, such as a failure to start a component or to complete an action.

    Open up Event Viewer > on the left side, open the Windows Logs folder > chose Application > wait for it to load > look for the most recent Error and click on it > see if it's related to Rainmeter.exe > click the area of the text and inspect it.

    Note that this is only orientative, most of the time, errors that appear here are those related to Out of Memory Access..

    To fix these errors, most people have reinstalled Rainmeter, reinstalled skins, reinstalled plugins or have run System File Checker.

  • Creating a crash dump

    • You can help the devs by creating a crash dump file. If you do, please:

    • Verify first that you are running the latest version of Rainmeter (try also the latest beta version if one is available)
    • Make sure that you are able to reproduce the crash
    • Check the forum to make sure that the crash hasn't been reported already (you can add more context or info if needed)
    • If the crash hasn't been reported earlier describe the exact steps how it can be reproduced
    • Write your system configuration (operating system, etc), what skins you are using and provide a download link to the dmp file

    If you do so, please report them in the Bugs & Feature Suggestions over on the Rainmeter Forum.
