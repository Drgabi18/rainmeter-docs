--- layout: tips title: 'Dynamic Cheat Sheet' ---

When you want to use the current value of a Variable, Built-in Variable, Section Variable, or Measure name anywhere in a Measure or Meter or Bang in Rainmeter, It is important to know when that value is automatically "dynamic" (always reflects the current value) or needs to be made "dynamic" with the DynamicVariables=1 option.

The following is a "cheat sheet" for the rules that apply in different situations.

Dynamic "Variables":

If you want to reference the value of a #VarName# [Variable] in a meter or measure, and that variable is dynamically set or changed with !SetVariable, you must use DynamicVariables on the meter or measure using it for it to stay "current". If the variable is "static", then DynamicVariables is not needed.

When using Built-in Variables, most of them are of a "static" nature and do not require DynamicVariables. Several of them, in particular Current Config size and position and Monitor Variables always require DynamicVariables to use the current values.

Dynamic "Measures":

If you want to reference the value of a measure in a meter or measure, then you must enclose the measure name in [] and have DyanmicVariables=1 on the meter or measure using it. This uses the string value of the measure as a Section Variable.

Exceptions:

  • If you are "binding" a measure to a meter with the MeasureName option, you don't need either [] or DynamicVariables.
  • If you are "binding" a parent WebParser measure to a child measure with the URL option, you need the [], but not DynamicVariables.
  • If you reference measures other than the parent in the URL option of a WebParser measure, a special [&MeasureName] Section Variable with [] and DynamicVariables must be used.

  • The Formula option in a Calc measure does not require either [] or DynamicVariables when referencing measures.
  • Measures used in IfCondition formulas are like the Formula option in a Calc measure, and do not require either [] or DynamicVariables. (Rainmeter 3.1 or later)

    Note that the Calc measure and IfCondition option are the only exceptions concerning formulas, and a measure used in a (formula) anywhere else will require both [] and DynamicVariables.

Dynamic "Bangs":

Measures in a (formula), Section Variables, Variables, or Built-in Variables used in any Bang do not require DynamicVariables. Things in Bangs are just always dynamic.

If the !SetOption bang is used to set Any Measure in a (formula), Section Variable, Variable, or Built-in Variable, the meter or measure that is being changed is automatically made dynamic for one update. If the !SetOption is done on every update, then DynamicVariables is not needed on that target meter or measure. If the !SetOption is not done on every update, then DynamicVariables will be needed on the target for it to stay "current".
