Two of the meters in Rainmeter use a unit of measurement called Radians in order to calculate a position on the circumference of a circle. This is a common unit used by mathematicians, scientists, and yes, programmers when working with things that are round. This guide will try to make it easier to understand, calculate and use radians in the context of Rainmeter.

The two meters that use radians in their options are Roundline and Rotator. Roundline draws a line that rotates around a center point based on some measure value. Rotator does much the same, using an image you provide rather than drawing a line. Both use StartAngle and RotationAngle to control the starting point on the circle, and the total amount of distance around the circle that constitutes 100 percent. That way they can be used to display the value of any measure (or formula) that defines a percentage.

So what are radians?

Without going too deeply into the math involved, (there are tons of sites like Wikipedia that can melt your brain with the details) radians measure a distance in degrees around the circumference of a circle. The default starting point, which is 0° or 360° degrees, is directly to the right of the center point. This is probably the most important thing to wrap your head around right from the start. Radians have nothing at all to do with the degrees on a compass, which is traditionally shown with 0° degrees (or North) pointing straight up.

So in short, radians are a way of defining a point on a circle, by starting at 0° degrees (directly to the right of center) and measuring the distance around the circle until you get to the point you want.

The method for calculating the number of radians to any point of a circle is pretty simple. It does make use of the mathematical constant PI and PI has the advantage of making these calculation work independant of the size of the circle.

You are unlikely to be able to think about the distance around a circle in terms of radians automatically. They are defined as very precise numbers with lots of decimal places, and in any case you would need to factor the mathematical constant PI into things, and ... let's just stop there. You can however think of a circle in terms of degrees. A circle has 360 degrees, and we are all used to and can easily picture what 45°, 90°, 180° and 270° mean in terms of distance. All we need to do is think about the circle with 0° pointing to the right of center, and add the number of degrees we need to get to the desired point.

So for example, if we want to define the point at the top of a circle, we can easily see that it will be 270° away from 0°, going clockwise around the circle. So what we want is a way to define that distance of 270° as a number of radians. This is done with a simple formula:

Degrees to Radians: (degrees*(PI/180))

You can also reverse the process if you need to:

Radians to Degrees: (radians*(180/PI))

Example

We can use a CPU meausre, which returns the amount of CPU usage as a percentage from 0% to 100%. Then we want to have a Roundline meter that displays the current value as filling a circle. We want the starting point at the top of the circle, and 100% defined as coming all the way back around to the top.

{% highlight ini %} [MeasureCPU] Measure=CPU [MeterCPUCircle] Meter=Roundline MeasureName=MeasureCPU W=40 H=40 StartAngle=((3*PI)/2) RotationAngle=(2*PI) LineWidth=5 LineStart=15 LineLength=20 LineColor=255,255,255,255 Solid=1 AntiAlias=1 {% endhighlight %}

The important bits here are the two options StartAngle and RotationAngle. StartAngle is defined as ((3*PI)/2), which is to say 270°. This means that if you start directly from the right of center at 0° / 360° it will take 270° degrees in radians to define the distance to the top of the circle moving clockwise. That will be the starting point for the meter. RotationAngle is defined as (2*PI), which is telling Rainmeter that the distance defined as 100% is all the way around 360°, or the full circle.

Note that in the context of StartAngle, 0° and 360° are really the same thing, as they both define the point on the circle that is directly to the right of center.

In another example, if you wanted to do a "VU Meter" type of approach, with the starting point on the left and 100% defined on the right, you would use:

{% highlight ini %} StartAngle=(PI) RotationAngle=(PI) {% endhighlight %}

If you look at the cheat-sheet above, you will see that this is saying start 180° clockwise from 0° / 360° (remember, starting directly right of center), and travel a total distance of 180° from there to define 100%.

A Rainmeter shortcut

While we think it is important to have a basic understanding of how radians work in Rainmeter, there is a way to cheat. There is a function we have created that can be used to automtically convert a number of degrees to a number of radians for you. It is the Rad(x) function, which will convert "x" as a number of degrees to the corresponding number of radians.

So in our first example above, we could also use:

{% highlight ini %} StartAngle=(Rad(270)) RotationAngle=(Rad(360)) {% endhighlight %}
